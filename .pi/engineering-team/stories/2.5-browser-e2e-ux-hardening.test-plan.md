# Test Plan: Story 2.5 — Browser e2e (Playwright) + UX hardening

**Story:** `.pi/engineering-team/stories/2.5-browser-e2e-ux-hardening.md`
**ADR:** `.pi/engineering-team/decisions/0007-browser-e2e-ux-hardening.md`
**Date:** 2026-06-03

## Approach

Playwright drives the real app against the seeded local strfry (`playwright.config.ts` `webServer` = `relay:up → seed → vite --port 8123`). Browser, port, and relay notes are environment-driven (see ADR 0007): NixOS needs `PLAYWRIGHT_CHROMIUM_PATH`; ports 8080/8081 are taken here so the harness uses 8123. The specs are the story's deliverable — they both **encode the Stories 4–6 flows** (regression coverage) and **drive the two new UX features** (RED until built).

The browser specs are excluded from the fast unit gate (`vite.config.ts` excludes `test/browser/**`) and from the nak e2e config; `npm test` and `npm run test:e2e` are unchanged.

## RED confirmation (2026-06-03, before implementation)

`npm run test:browser` (with `PLAYWRIGHT_CHROMIUM_PATH` set): **5 passed, 2 failed.** The two failures are the unbuilt features:
- `login.spec.ts:4` — no visible login control in the header (only inside the closed Post-a-project dialog).
- `assign-arbiter.spec.ts:15` — no assign/change-arbiter control on the task detail page.

The regression-guard and the existing-flow specs pass — confirming (a) the harness drives the real app correctly and (b) the reported "No curators found" regression is **not reproducible on current code** (it is handled by `RelayEnvOverride`; the guard locks that in). If it ever recurs on current code, reproduce with the failing tab's `localStorage` (`nostr:app-config`, `nostr:login`, `grantless:lastCurator`).

## Test infrastructure

- Runner: `@playwright/test` (Chromium headless). Config: `playwright.config.ts` (port 8123, `executablePath` via `PLAYWRIGHT_CHROMIUM_PATH`, `webServer` reusing the seed).
- Helpers: `test/browser/helpers.ts` — `loginAs` (inject the `nostr:login` shape; decoupled from the login UI), `rememberCurator`, `selectCurator`; imports the fixed roster from `test/seed/accounts.ts`.
- Unchanged: `npm test` (unit) and `npm run test:e2e` (nak) — `test/browser/**` excluded from both.

## How to run

```
# NixOS: export PLAYWRIGHT_CHROMIUM_PATH=<a playwright-driver chrome>
npm run test:browser
```

## Verification

The two RED specs fail because the header login control and the detail-page arbiter control don't exist yet; they pass once Implementation adds `<LoginArea>` to the browse header and mounts `AssignArbiterControl` on `TaskDetail`. The 5 green specs continue to pass (regression coverage).
