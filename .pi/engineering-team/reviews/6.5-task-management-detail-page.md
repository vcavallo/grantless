## Things tests can't catch ŌĆö and three real bugs the browser tier surfaced

This story is the payoff for adding Playwright: driving the real UI exposed three genuine defects (not test noise), each now fixed and guarded:
1. **`TaskDetail` queried `authors:[patron]`** ŌĆö a worker/arbiter-signed status update is a replaceable event at a *different* coordinate, so submitted/concluded transitions were invisible on the detail page. Fixed: query by `#d` only + `latestAuthoritativeTask` (drops the filter, applies the authorized-updater rule across signers). Unit-tested (worker-signed wins; unauthorized signer rejected).
2. **Seed `created_at` overshoot** ŌĆö the harness monotonic clock drifted up to ~39s ahead of wall time, so a real-time transition on a freshly-seeded task *lost* the latest-wins race (a real manual-testing footgun, not just a test issue). Fixed: `anchorClock` backdates seeded events ~1h; seed e2e still green.
3. **Mock 9735 `e` tag held the a-coord** (`33401:ŌĆ”`) instead of a 32-byte hex event id ŌåÆ strfry rejects it (`OK:false`); nak silently ignored the rejection in the nak e2e, but the browser's nostrify surfaced it as the "All promises were rejected" toast the user-style flow hit. Fixed: reference the task event id. Also strengthened `useNostrPublish` retry to a short backoff sequence; the Playwright webServer now starts from a clean relay.

Other checks: no secrets/debug logging/placeholder comments in the diff; the acting user signs their own 33401/3402/9735 (no nsec exposure); `latestAuthoritativeTask` enforces the authorized-updater rule, so a stranger can't hijack a task's displayed status by reusing its `d`.
