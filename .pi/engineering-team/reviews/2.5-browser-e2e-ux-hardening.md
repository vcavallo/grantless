## Findings

### Blocking
None.

### Non-blocking
1. **Regression not reproduced on current code.** The fix is "already handled by `RelayEnvOverride`; now guarded." If the user still hits "No curators found" on a freshly-restarted `npm run dev`, the next step is their failing tab's `localStorage` (`nostr:app-config`, `nostr:login`, `grantless:lastCurator`) to reproduce exactly — not a blind change. Documented in ADR 0007 + test plan.
2. **NixOS-only browser caveat:** `npm run test:browser` needs `PLAYWRIGHT_CHROMIUM_PATH` here (Playwright's bundled Chromium lacks NixOS libs). CI/cross-platform provisioning is deferred (out of scope).
3. **Detail-page arbiter options depend on the remembered curator** — if a user lands on a task detail page without having browsed a curator first, the control hints to pick one. Acceptable for v1; a curator picker on the detail page is a possible later refinement.
